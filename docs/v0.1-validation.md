# v0.1 Validation

Use this checklist to manually validate A11y Agent Lab v0.1 Snapshot Lab.

1. Start the fixture app.

   ```bash
   npm run fixture:checkout
   ```

2. Run snapshot for `good-a11y`.

   ```bash
   npm run build
   npm run cli -- snapshot \
     --url "http://localhost:4310/checkout?variant=good-a11y" \
     --out "results/snapshots/manual-good"
   ```

3. Confirm all expected files exist.

   ```text
   screenshot.png
   metadata.json
   cdp-ax.json
   cdp-ax-summary.json
   aria.yml
   aria-summary.json
   dom-compact.json
   dom-summary.json
   ```

4. Run snapshot-suite.

   ```bash
   npm run cli -- snapshot-suite \
     --config "experiments/checkout.snapshot.yaml" \
     --out "results/snapshots"
   ```

5. Confirm each variant has an output directory.

   ```text
   results/snapshots/checkout/good-a11y/
   results/snapshots/checkout/no-label/
   results/snapshots/checkout/icon-only-button/
   results/snapshots/checkout/duplicate-names/
   results/snapshots/checkout/hidden-noise/
   ```

6. Confirm `summary.json` includes all variants.

   ```bash
   node -e "const s=require('./results/snapshots/checkout/summary.json'); console.log(s.variantCount, s.variants.map(v => v.id).join(','))"
   ```

7. Confirm `results.csv` has one row per variant.

   ```bash
   wc -l results/snapshots/checkout/results.csv
   ```

   The expected count is 6 lines: one header plus five variant rows.

8. Compare `good-a11y` and `icon-only-button` summaries.

   Check `cdp-ax-summary.json` and `results.csv` for unnamed interactive node differences.

9. Compare `good-a11y` and `hidden-noise` summaries.

   Check `dom-summary.json` and `results.csv` for hidden element count differences.

10. Run automated validation.

   ```bash
   npm run build
   npm test
   ```
